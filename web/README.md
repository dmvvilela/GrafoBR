# GrafoBR — web

Frontend: Next.js (App Router) + TypeScript + Tailwind v4 + D3 force graph.
Consumes the data contract (`../contract/`) and nothing else. No backend in v1.

## Run

```bash
pnpm install
pnpm dev          # http://localhost:3000
```

`pnpm typecheck` / `pnpm build` before shipping.

> Versions in `package.json` are caret ranges valid at scaffold time. If `pnpm install`
> complains, run `pnpm up --latest` and adjust.

## What's here

```
src/
  app/
    page.tsx                 # directory/search over the generated politician index
    politico/[id]/page.tsx   # SSG per-politician page
    layout.tsx, globals.css
  components/
    NetworkGraph.tsx         # interactive d3-force ego-network graph
    EgoView.tsx              # graph page shell + selected-node details
    SearchDirectory.tsx      # Fuse.js client search over the index
  lib/
    contract.ts              # TS mirror of the data contract (the seam)
    data.ts                  # build-time static JSON reader
    graph-colors.ts          # category/edge color maps (ready to use)
public/
  data/                       # generated by scripts/sync-data.mjs before dev/build
```

## Data flow
The frontend only ever reads **static JSON files**. `scripts/sync-data.mjs` copies
`../data/*.json` into `public/data/` before `pnpm dev` and `pnpm build`, then Next.js
pre-renders one `/politico/[id]` page per entry in `index.json`.

The web never queries a database or runtime API — see `../docs/DECISIONS.md` D1/D6.
